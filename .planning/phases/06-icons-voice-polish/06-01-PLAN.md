<task type="auto">
  <name>Task 1: Install lucide-react and create Icon component</name>
  <files>components/shared/Icon.tsx</files>
  <action>
Install lucide-react:
```
npm install lucide-react
```

Create components/shared/Icon.tsx as a thin wrapper that re-exports ~20 named Lucide icons
as a single unified component. Use the discriminated-union name pattern so tree-shaking
works correctly with optimizePackageImports already configured in next.config.js.

Exported union type IconName must cover exactly these names:
  book, calendar, file-text, plus, arrow-right, arrow-left, search, check, x, clock,
  sparkles, upload, graduation-cap, chevron-right, link, flag, layers, flask, brain, chevron-left

Component signature:
```typescript
export type IconName =
  | "book" | "calendar" | "file-text" | "plus" | "arrow-right" | "arrow-left"
  | "search" | "check" | "x" | "clock" | "sparkles" | "upload"
  | "graduation-cap" | "chevron-right" | "link" | "flag" | "layers"
  | "flask-conical" | "brain" | "chevron-left";

interface IconProps {
  name: IconName;
  size?: number;      // default 16
  className?: string;
}

export function Icon({ name, size = 16, className }: IconProps): JSX.Element
```

Map each IconName string to its Lucide import. Import each icon by name from "lucide-react"
at the top of the file (named imports — not dynamic, so tree-shaking applies at build time).

Render each as `<LucideIcon size={size} strokeWidth={1.5} className={className} />`.

The component should be a simple switch/map — no dynamic import(), no lazy loading.
Lucide icons already carry aria-hidden by default; no extra ARIA needed.

Use `currentColor` behavior inherited from Lucide defaults — do NOT set any stroke or fill color directly.
  </action>
